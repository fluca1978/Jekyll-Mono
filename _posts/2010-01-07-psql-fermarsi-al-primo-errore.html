Il terminale interattivo di PostgreSQL, psql, consente di eseguire un file di comandi SQL in sequenza con il comando speciale \i nomeFile. Tuttavia se tale file contiene degli errori psql continuer√† a processare il file (anche se i comandi non verranno applicati a causa del fallimento della transazione corrente). Come risultato si vedranno passare sullo schermo una serie di messaggi di errore causati dal primo errore rilevato nel file. E' possibile dire a psql di fermarsi al primo errore rilevato nel file mediante il comando speciale

\set ON_ERROR_STOP


da lanciare prima dell'esecuzione del file comandi tramite \i.
